# RTO Management System – Project Documentation

## 1. Project Overview

The RTO (Return to Office) Management System is a web-based portal for organizations to track and manage employees’ in-office, work-from-home, and leave statuses. Employees can update their daily timesheets, and managers can download consolidated reports for their teams.

---

## 2. Technology Stack

- **Backend:** Python Flask (REST API)
- **Frontend:** Flask with Jinja2 templates
- **Database:** PostgreSQL (robust, scalable, and well-supported on Azure)
- **Deployment:** Azure App Service
- **CI/CD:** Azure DevOps Pipeline

---

## 3. Database Design

### Table: employee_attendance

| Column Name   | Type         | Description                        |
|---------------|--------------|------------------------------------|
| id            | SERIAL (PK)  | Unique record ID                   |
| emp_id        | VARCHAR      | Employee ID                        |
| emp_name      | VARCHAR      | Employee Name                      |
| manager_id    | VARCHAR      | Manager’s Employee ID              |
| date          | DATE         | Date of attendance                 |
| status        | CHAR(1)      | 'Y' (WFO), 'N' (WFH), 'L' (Leave)  |
| hours         | FLOAT        | Hours worked (if applicable)       |
| submitted     | BOOLEAN      | Timesheet submitted or not         |

### Table: employees

| Column Name   | Type         | Description                        |
|---------------|--------------|------------------------------------|
| emp_id        | VARCHAR (PK) | Employee ID                        |
| emp_name      | VARCHAR      | Employee Name                      |
| manager_id    | VARCHAR      | Manager’s Employee ID              |
| email         | VARCHAR      | Employee Email                     |
| password_hash | VARCHAR      | Hashed password                    |
| role          | VARCHAR      | 'employee' or 'manager'            |
